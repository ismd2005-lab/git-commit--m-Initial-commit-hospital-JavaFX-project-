1. INTRODUCTION

Dans le cadre du mini projet GI3, nous avons realise une application desktop de gestion medicale developpee avec JavaFX. Le projet consiste a concevoir et implementer une solution permettant a une clinique de gerer ses patients, ses rendez-vous, ses statistiques et ses exports de donnees.

L'application porte le nom MedVision AI Healthcare Dashboard. Elle adopte une interface moderne et ergonomique, avec un tableau de bord central permettant de visualiser rapidement les informations importantes de la clinique. Le projet repose sur une architecture MVC, une base de donnees MySQL et une organisation claire du code en modeles, DAO, controleurs et vues FXML.


2. PRESENTATION DU PROJET

MedVision AI Healthcare Dashboard est une application desktop destinee a faciliter la gestion quotidienne d'une clinique medicale.

Elle permet principalement de :
- gerer les dossiers des patients ;
- ajouter, modifier, supprimer et rechercher des patients ;
- planifier et suivre les rendez-vous medicaux ;
- filtrer les donnees selon plusieurs criteres ;
- afficher des statistiques sur les patients et les rendez-vous ;
- exporter les donnees au format CSV ;
- personnaliser l'apparence de l'application.

L'objectif principal du projet est de centraliser les informations medicales essentielles dans une interface simple, claire et efficace.

4. CONCEPTION DE L'APPLICATION

4.1 Architecture generale

Le projet suit l'architecture MVC : Modele, Vue, Controleur.

Modele :
Les classes Patient et RendezVous representent les donnees principales de l'application.

Vue :
Les interfaces sont definies dans des fichiers FXML :
- dashboard.fxml ;
- patients.fxml ;
- rendezvous.fxml.

Controleur :
Les controleurs Java gerent les actions utilisateur et la logique d'affichage :
- DashboardController ;
- PatientController ;
- RendezVousController.

DAO :
Les classes DAO assurent la communication avec la base de donnees :
- PatientDAO ;
- RendezVousDAO ;
- Database.

Cette organisation permet de separer clairement les responsabilites et de rendre le code plus lisible et maintenable.

5.3 Gestion de la connexion a la base de donnees

La classe Database contient la configuration de connexion a MySQL. Elle utilise JDBC pour etablir la connexion avec la base medvision_ai.

La connexion peut etre personnalisee a l'aide de variables d'environnement :
- MEDVISION_DB_URL ;
- MEDVISION_DB_USER ;
- MEDVISION_DB_PASSWORD.

Si aucune variable n'est definie, l'application utilise une configuration par defaut pointant vers une base locale MySQL.

5.4 Gestion des patients

La gestion des patients est assuree par PatientController et PatientDAO.

PatientController gere :
- la saisie des informations ;
- la validation des champs ;
- l'affichage dans le tableau ;
- la recherche dynamique ;
- les filtres ;
- les actions ajouter, modifier et supprimer.

PatientDAO gere :
- l'insertion d'un patient ;
- la modification d'un patient ;
- la suppression d'un patient ;
- la recuperation des patients ;
- les recherches avec criteres ;
- les statistiques liees aux patients.

Une validation est effectuee sur certains champs, notamment l'adresse email.

5.5 Gestion des rendez-vous

La gestion des rendez-vous est assuree par RendezVousController et RendezVousDAO.

RendezVousController permet :
- de selectionner un patient ;
- de choisir une date ;
- de saisir une heure ;
- de choisir une specialite ;
- de selectionner un medecin ;
- de definir un statut ;
- d'ajouter une remarque ;
- de rechercher et filtrer les rendez-vous.

Une validation est appliquee au format de l'heure, qui doit respecter le format HH:mm.

RendezVousDAO realise les operations SQL necessaires et utilise une jointure entre RENDEZVOUS et PATIENT afin d'afficher le nom complet du patient dans la table des rendez-vous.

5.6 Export CSV

La classe CsvExporter permet d'exporter les donnees affichees dans les tables vers un fichier CSV.

Elle est utilisee pour :
- exporter les patients ;
- exporter les rendez-vous ;
- exporter un resume du dashboard.

L'export utilise FileChooser afin de laisser l'utilisateur choisir l'emplacement du fichier.

7. BASE DE DONNEES

7.1 Presentation

La base de donnees utilisee est medvision_ai. Elle contient deux tables principales :
- PATIENT ;
- RENDEZVOUS.

7.2 Table PATIENT

La table PATIENT contient les informations des patients.

Champs principaux :
- id_patient : identifiant unique du patient ;
- nom : nom du patient ;
- prenom : prenom du patient ;
- age : age du patient ;
- sexe : sexe du patient ;
- telephone : numero de telephone ;
- email : adresse email ;
- description : informations medicales ;
- date_creation : date de creation du dossier ;
- actif : statut du patient.

7.3 Table RENDEZVOUS

La table RENDEZVOUS contient les rendez-vous medicaux.

Champs principaux :
- id_rdv : identifiant unique du rendez-vous ;
- id_patient : identifiant du patient concerne ;
- date_rdv : date du rendez-vous ;
- heure : heure du rendez-vous ;
- medecin : medecin responsable ;
- specialite : specialite medicale ;
- statut : etat du rendez-vous ;
- remarque : remarque complementaire.

7.4 Relation entre les tables

La relation entre PATIENT et RENDEZVOUS est une relation de type un-a-plusieurs.

Un patient peut avoir plusieurs rendez-vous. Un rendez-vous appartient a un seul patient.

La cle etrangere id_patient dans la table RENDEZVOUS reference id_patient dans la table PATIENT.

L'option ON DELETE CASCADE permet de supprimer automatiquement les rendez-vous lies lorsqu'un patient est supprime.
